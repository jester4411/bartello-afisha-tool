# Bartello Afisha Tool

Внутренний веб-инструмент для команды Bartello. Автоматизирует подготовку CSV-файлов для загрузки в систему управления афишей поездов.

## Стек

Одностраничное приложение — один файл `index.html` (HTML + CSS + JS). Без зависимостей, без сборки, без сервера. Деплой через GitHub Pages.

## Как запустить локально

Открыть `index.html` в браузере.

## Деплой

GitHub Pages из ветки `main`. Ссылка: `https://jester4411.github.io/bartello-afisha-tool/`

Для обновления — запушить изменения в `main`, Pages обновится автоматически.

## Три шага работы

### Шаг 1 — Генерация афиши

- Вход: CSV расписания (название, время отправления, дата начала, дата окончания, периодичность, периоды, время окончания для предзаказа) + код поезда
- Если дата окончания пустая — создаётся одно мероприятие на дату начала
- Периодичность: `каждый день` (шаг 1) или `через день` (шаг 2)
- Выход: `afisha_{код}.csv`

### Шаг 2 — Матчинг ID

- Вход: файл афиши из шага 1 + выгрузка ID мероприятий из системы (разделитель `;`, UTF-8 с BOM, даты на русском)
- Нормализация названий: убираются пробелы вокруг дефисов
- Выход: `afisha_{код}_with_ids.csv`

### Шаг 3 — Генерация периодов

- Вход: файл афиши с ID из шага 2 + исходный CSV расписания
- Периоды (слоты) берутся из колонки `периоды` исходного CSV
- Время финального заказа берётся из колонки `время окончания для предзаказа`: `Ч:ММ` — до начала, `+Ч:ММ` — после начала
- Выход: `periods_{код}.csv`

## Важные детали

- Все выходные файлы: UTF-8 без BOM, разделитель запятая
- Час без ведущего нуля в выходных файлах (`5:40:00`, не `05:40:00`)
- Колонка `периоды` — многострочная (разбивается по `\n`)
- Язык интерфейса — русский
